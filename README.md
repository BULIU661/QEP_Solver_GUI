# QEP Solver — 二次特征值问题求解器（GUI 完整版）

基于 **Eigen** 和 **Spectra** 的稀疏矩阵二次特征值问题求解器的图形界面版本。通过可视化面板配置参数、选择问题、运行求解、查看结果，无需编辑 JSON 或使用命令行。

## 项目结构

```
QEP/
├── bin/qep_gui               GUI 可执行文件
├── config.json               运行时配置文件（与 CLI 共享）
├── Problems/                 测试问题数据（.bin / .txt）
├── Results                   gui求解结果导出目录
├── src/gui/                  GUI 源代码
│   ├── main.cpp              入口
│   ├── MainWindow.cpp        主窗口
│   ├── panels/               配置面板、输出面板、问题树、求解器选择
│   └── workers/              后台求解线程
└── src/qep/                  核心求解器库（GUI 不直接依赖）
```

## 启动 GUI

```bash
# 构建（需 Qt5 或 Qt6）
cmake .. -DQEP_BUILD_GUI=ON
cmake --build . -j8

# 启动
./bin/qep_gui
```

启动后自动加载 `config.json`，所有参数与 CLI 版本共享同一份配置文件。

## 操作流程

### 第 1 步：选择问题

在左侧**问题树**中勾选要求解的用例。问题按分组（benchmark / engineering / small_demo / user）分类显示。

- **右键菜单**：新建问题、删除问题、转换为二进制格式、在文件管理器中打开
- **拖放支持**：可拖入外部矩阵文件
- **自动刷新**：文件系统中的变化会自动同步到问题树

### 第 2 步：选择求解器

在左侧**求解器面板**中勾选要使用的线性求解器。上半部分是直接法（PardisoLU / SparseLU / SimplicialLLT），下半部分是迭代法（CG / BiCGSTAB / GMRES）。

每次勾选/取消会立即写入 `config.json` 的 `enabled_solvers` 字段。

### 第 3 步：配置参数

切换到右侧的**参数配置**选项卡，4 个子页面：

**求解设置**：特征值个数（NEV）、位移点（σ）、Arnoldi 容差和迭代次数、残差检验容差。预设下拉框提供"标准"、"高精度"、"快速"三种快速配置。

**线性求解器**：内层容差、内层最大迭代、ILUT 填充因子/丢弃容差、GMRES 重启步数。

**性能优化**：OpenMP 线程数、MKL 线程数、自适应参数开关。修改线程数实时生效。AVX2 状态为只读（由编译选项决定）。

**文件转换**：`auto_convert`（自动 .txt → .bin 转换）和 `overwrite`（覆盖已有 .bin）开关。

### 第 4 步：运行求解

勾选问题和求解器后，点击侧边栏底部的**"求解选中问题"**按钮。求解在后台线程中异步运行，不会阻塞界面。

- **进度条**显示当前进度
- 求解过程中可随时点击**停止**中断

### 第 5 步：查看结果

求解完成后自动切换到**求解输出**选项卡，显示两个子页面：

**结果表格**（默认显示）：

| 列 | 内容 |
|---|---|
| `#` | 特征值序号 |
| `特征值` | 实部/虚部 |
| `|λ|` | 特征值模长 |
| `相对残差` | 相对残差（科学计数法） |
| `绝对残差` | 绝对残差（科学计数法） |
| `状态` | 4 级评定：EXCELLENT / OK / ACCEPTABLE / WARNING |

- **双击表格行**查看该结果的完整详情（概览卡片 + 全量特征值表 + 残差详情 + 性能统计）
- **导出按钮**：支持 JSON 和 CSV 两种格式导出
- **摘要栏**：表格上方显示最新结果的概览（用例名 / 求解器 / 特征值数 / 收敛状态 / 最大残差）
- 多结果集之间有分隔行区分

**文本输出**：完整的求解日志，包含环境信息、矩阵属性、每个求解器的运行细节和性能统计。

- **过滤栏**：输入关键字高亮匹配行
- **保存按钮**：将全部日志保存为 `.txt` 文件
- **清空按钮**：清空日志和所有结果

## 菜单功能

**文件菜单**：
- **加载配置**：从外部 JSON 文件加载参数配置
- **保存配置**：将当前界面参数保存到 `config.json`
- **退出**：关闭程序

## 与 CLI 版本的关系

| 特性 | CLI (`qep`) | GUI (`qep_gui`) |
|---|---|---|
| 配置文件 | 同一个 `config.json` | 同一个 `config.json` |
| 参数修改 | 编辑 JSON 或命令行 flag | 界面控件直接改 |
| 问题选择 | `active_cases` 或 `--case` | 问题树勾选 |
| 求解器选择 | `enabled_solvers` 字段 | 求解器面板勾选 |
| 结果查看 | 终端格式化表格 | 可排序表格 + 详情面板 |
| 结果导出 | 无 | JSON / CSV |
| 批量运行 | 多个用例依次求解 | 多个用例依次求解 |
| 适用场景 | CI / 脚本 / 服务器 | 交互式探索 / 调参 |

两个版本**共享同一份 `config.json`**——在 GUI 中调的参数可以直接用 CLI 复现，反之亦然。
